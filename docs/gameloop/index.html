Game Loop

Inside the Engine2D class there's a Game Loop that handles the input, runs the Physics Engine to update every object and renders everything to the screen. Right before this last rendering, the Engine execute what you, the developer, have written into your code. In fact setting up your Game Loop is really easy. The Engine2D class has a method, called loop that takes in as a parameter a GameLoop object, where you define your loop. I know, the word "loop" is used too much. Furthermore you can define what the Engine should execute before the loop (and so the game) starts and after it ends, before shutting everything down. Let's give a look to the simple code. 


engine.loop(new GameLoop()
{
	@Override
	public void start()
	{
		// What you write here will be executed before the game to start
	}
	
	@Override
	public void run()
	{
		// What you write here will be executed every frame
	}
	
	@Override
	public void stop()
	{
		// What you write here will be executed after the game to end, before closing the program.
	}
});

As I promised the code is in fact really really simple. That's because I do not want you to spend too much time on aspects different from the game logic. You handle it, the Engine handle everything else. The final code will look something like this: 


import org.lorenzoleonardini.badger.Engine2D;
import org.lorenzoleonardini.badger.GameLoop;

public class HelloWorld
{
	public static void main(String[] args)
	{
		// Create and initialize the 2D engine
		Engine2D engine = new Engine2D();

		// Create the window. Parameters are width, height, scale and title
		engine.createWindow(900, 516, 1, "Hello World!");

		// Set up the Game Loop
		engine.loop(new GameLoop()
		{
			@Override
			public void start()
			{
				/* ------------ What you write here will be executed before the game to start ------------ */
			}
			
			@Override
			public void run()
			{
				/* ------------------- What you write here will be executed every frame ------------------ */
			}
			
			@Override
			public void stop()
			{
				/* What you write here will be executed after the game to end, before closing the program. */
			}
		});
		
		// Start the engine
		engine.start();
		
		engine.logFPS = false;
		engine.logObjects = false;
	}
}

That's it! Really easy, isn't it?

Legitimate questions

Are there any differences between a code put into the start method and a code put right before engine.start()?
Of course yes. engine.start() initialize the Engine and all its features, and it also loads every resources from files. The start method is executed after these initializations and allows you to do whatever you want. On the contrary, code written before engine.start() is executed before the Engine to start: every code that uses Engine components or resources from files will cause a bunch of errors.

So, are there any differences between a code put into the start method and a code put AFTER engine.start()?
In this case... I don't know. Giving a first look... no... but because the Engine runs in a separate Thread I don't suggest to do like that. Use the start method. It's there for a reason.
