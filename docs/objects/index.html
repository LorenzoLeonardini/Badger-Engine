Objects

Every single object in your game must correspond to a PhysicsObject in the Engine. PhysicsObjects are what can be rendered and updated by the PhysicsEngine.

Creating them is really easy. The constructor takes in a bunch of parameters: the first two are the x and y coordinates in the world, than we have an ObjectRender and a Material. The ObjectRender is what is actually rendered by the Engine; it can be a Shape, like a Circle or a Rectangle, or a Texture. The Material is a class used by the PhysicsEngine to handle collisions. Let's continue creating a Circle c with a radius of 20 and a Material m. The Material constructor needs the material density as a parameter. For now let's set it to 1. The code will be 


Circle c = new Circle(20);
Material m = new Material(1);
PhysicsObject obj = new PhysicsObject(450, 250, c, m);

As default every PhysicsObject is affected by gravity, can move and be moved and can collide with every other object. All these settings can be changed by changing the value of the following boolean variables: 


// Change if the object is affected by gravity
obj.setAffectedByGravity(false);
// Change if the object can move or be moved
obj.setUnmovable(true);
// Change if the object can collide. In this particular case this is a property of its material
m.setTransparent(true);

Let's talk about the life of our object.
The quickest way to destroy an object is "killing" it. However we can also set a lifespan, in milliseconds.

IMPORTANT: for debugging reasons every object is destroyed when its y coordinate is greater than window's height. You can turn off this "feature". 


// Kill the object
obj.kill();
// Set a lifespan
obj.setLifespan(2500);
// Turning of automatic killing y > widow height
obj.setKillWhenOutOfBounds(false);

There's a last thing to mention. Remember the object counter logged in the console? It tells you how many objects are still alive and so processed by the PhysicsEngine. This feature allows you to check if you are keeping alive useless objects that have the only scope of slowing everything down.

Give a look to the full code: 


import org.lorenzoleonardini.badger.Engine2D;
import org.lorenzoleonardini.badger.GameLoop;
import org.lorenzoleonardini.badger.physics.Material;
import org.lorenzoleonardini.badger.physics.PhysicsObject;
import org.lorenzoleonardini.badger.texture.Circle;

public class HelloWorld
{
	public static void main(String[] args)
	{
		// Create and initialize the 2D engine
		Engine2D engine = new Engine2D();

		// Create the window. Parameters are width, height, scale and title
		engine.createWindow(900, 516, 1, "Hello World!");

		// Set up the Game Loop
		engine.loop(new GameLoop()
		{
			@Override
			public void start()
			{
				/* ------------ What you write here will be executed before the game to start ------------ */
				
				Circle c = new Circle(20);
				Material m = new Material(1);
				PhysicsObject obj = new PhysicsObject(450, 250, c, m);
				
				// Change if the object is affected by gravity
				obj.setAffectedByGravity(false);
				// Change if the object can move or be moved
				obj.setUnmovable(true);
				// Change if the object can collide. In this particular case this is a property of its material
				m.setTransparent(true);
				
				// Set a lifespan
				obj.setLifespan(2500);
				// Turning of automatic killing y > widow height
				obj.setKillWhenOutOfBounds(false);
			}
			
			@Override
			public void run()
			{
				/* ------------------- What you write here will be executed every frame ------------------ */
			}
			
			@Override
			public void stop()
			{
				/* What you write here will be executed after the game to end, before closing the program. */
			}
		});
		
		// Start the engine
		engine.start();
		
		engine.logFPS = false;
		engine.logObjects = false;
	}
}
